# L0 · Triage Pass（原料分流）

## 目的

`raw/` 下的学习材料往往是混合的——同一个文件里可能同时包含工具使用说明、领域技能描述和其他内容。混合材料会稀释训练信号。

Triage Pass 在任何 Lx 语言分析启动之前，对 `raw/` 下的文件**按内容类型分流**，将不同训练信号分离到各自的目标文件，使后续 L-Pass 作用于纯净的单类材料。

## 内容类型定义

所有 L0 分类**必须基于模型语义判断**，不得以关键字命中作为分类依据。

L0 的分流单位是**句子**；当多个相邻句子共同表达同一语义功能时，以**句组**为单位处理。

### Expert（领域技能类）

描述**具体领域的操作方法或最佳实践**的文字——即"怎么做某件具体的事"。判断标准：

- 描述特定技术栈的操作步骤（如怎么写 React Component、怎么建 Mongo Connection）
- 提供领域规范、模式或反模式
- 包含具体的代码片段，且该代码展示的是领域知识而非工具用法

**不属于 Expert**：通用写作规则、Steering 语言规律、工具调用方式、情绪风格描述、行为约束。

### Tools-TodoList（任务列表类）

描述 Agent **将当前任务自主拆分为本地任务列表、逐项执行并追踪完成状态**的文字。判断标准：

- 描述 Agent 主动创建任务列表、分解子任务的机制
- 描述任务状态的生命周期管理（创建 → in_progress → completed / accomplished）
- 说明何时启动列表、如何逐条推进、如何关闭整个列表
- 包含具体实现此机制的工具（如 `todo_write`）的详细用法说明

**不属于 Tools-TodoList**：仅一句话提及工具名称的工具定义行；描述 Agent 整体顺序执行风格的约束句（那属于 Constraints）。

### Tools（工具类）

描述**有哪些工具可用、如何调用、调用参数、使用示例**的文字。判断标准：

- 提及具体工具名称并说明其用途或用法
- 描述工具的输入/输出、参数签名
- 提供工具调用的示例代码或伪代码

**不属于 Tools**：仅提到工具名称作为概念引用、未涉及用法的句子。若某些能力描述在 Tools 阶段未被判断为工具相关，则继续流向后续步骤，不在此阶段强行归类。

### Emotional（情绪/风格类）

描述 Agent 的**情绪、态度、语气、风格、人格化呈现方式**的文字。判断标准：

- 描述 Agent 应该以什么语气、口吻或感觉出现
- 描述 tone、voice、style、personality 一类表达要求
- 描述 warm、friendly、supportive、optimistic 等情绪或态度取向

**不属于 Emotional**：与做事直接相关的角色、任务定位、工具用法、行为约束。

### Constraints（约束类）

描述 Agent **被允许、被禁止、必须如何行动**的文字。判断标准：

- 描述 must、must not、never、do not、cannot、only 等约束要求
- 描述权限边界、安全边界、输出边界、流程边界
- 描述禁止事项、必守规则、强制格式或强制步骤

**不属于 Constraints**：纯工具说明、纯领域做法、纯情绪风格描述。

### Unclassified（暂不分类）

不属于上述四类，或语义判断存在不确定性的内容。**原地保留**，不做移动。

## 触发条件

以下任一情况出现时，MUST 先执行 Triage Pass：

- 新的学习材料被加入 `raw/` 目录
- Lx Pass 发现材料混合导致扫描结果杂乱
- 明确被要求对 `raw/` 执行整理

## 执行步骤

### 步骤 1：目录扫描

列出 `raw/` 下的所有子目录。**按目录为单位**逐一处理，一个目录完成后再进入下一个。

### 步骤 2：句子/句组级串行分类

对当前目录下每个 `.md` 文件，按 **`Expert -> Tools-TodoList -> Tools -> Emotional -> Constraints`** 的顺序，对文本做**逐句语义分析**；当多个相邻句子共同表达同一语义功能时，以**句组**为单位处理。

- `[Expert]`
- `[Tools-TodoList]`
- `[Tools]`

记录每个来源文件在各分类下的句组数量，用于步骤 5 的汇总统计。

遇到判断模糊的句子或句组：优先标 `[Unclassified]`，不强行归类。

### 步骤 3：汇总统计

将步骤 2 的分类结果以目录为单位汇总为统计结果，继续执行当前目录的分流。

### 步骤 4：执行分流

对当前目录执行：

1. **创建或追加** `{目录}/Expert.md`：
   - 将本目录内所有来源文件中标记为 `[Expert]` 的句子或句组**按来源文件分节**写入
   - 每节头部注明来源：`## 来源：{原文件名}`
   - 若 `Expert.md` 已存在，**追加**而非覆盖

2. **创建或追加** `{目录}/Tools-TodoList.md`：
   - 将本目录内所有来源文件中标记为 `[Tools-TodoList]` 的句子或句组**按来源文件分节**写入
   - 每节头部注明来源：`## 来源：{原文件名}`
   - 若 `Tools-TodoList.md` 已存在，**追加**而非覆盖

3. **创建或追加** `{目录}/Tools.md`：
   - 将本目录内所有来源文件中标记为 `[Tools]` 的句子或句组**按来源文件分节**写入
   - 每节头部注明来源：`## 来源：{原文件名}`
   - 若 `Tools.md` 已存在，**追加**而非覆盖

4. **创建或追加** `{目录}/Emotional.md`：
   - 将本目录内所有来源文件中标记为 `[Emotional]` 的句子或句组**按来源文件分节**写入
   - 每节头部注明来源：`## 来源：{原文件名}`
   - 若 `Emotional.md` 已存在，**追加**而非覆盖

5. **创建或追加** `{目录}/Constraints.md`：
   - 将本目录内所有来源文件中标记为 `[Constraints]` 的句子或句组**按来源文件分节**写入
   - 每节头部注明来源：`## 来源：{原文件名}`
   - 若 `Constraints.md` 已存在，**追加**而非覆盖

6. **删除原文件中已移走的句子或句组**：
   - 原文件保持文件名不变
   - 仅删除被分类为 `[Expert]`、`[Tools-TodoList]`、`[Tools]`、`[Emotional]`、`[Constraints]` 的句子或句组
   - `[Unclassified]` 内容原地保留
   - 若一个段落中只有部分句子被移走，仅删除对应句子或句组，不得连带删除相邻的无关句子
   - 原文件剩余内容的顺序和结构不做任何其他变更

### 步骤 5：完成报告

```

### 处理目录：{目录名}

| 来源文件 | 移走 Expert 句组 | 移走 TodoList 句组 | 移走 Tools 句组 | 移走 Emotional 句组 | 移走 Constraints 句组 | 保留 Unclassified 句组 |
|---------|------------------|--------------------|-----------------|----------------------|------------------------|------------------------|
| Gpt.md  | 1                | 1                  | 3               | 2                    | 1                      | 7                      |
| Opus.md | 0                | 0                  | 2               | 1                    | 0                      | 5                      |

### 输出文件
- `Expert.md`：已创建 / 已追加（共 N 个句组）
- `Tools-TodoList.md`：已创建 / 已追加（共 N 个句组）
- `Tools.md`：已创建 / 已追加（共 N 个句组）
- `Emotional.md`：已创建 / 已追加（共 N 个句组）
- `Constraints.md`：已创建 / 已追加（共 N 个句组）

### 原文件变更
- `Gpt.md`：删除 7 个句组，剩余 7 个句组

```

## 停止条件

当前目录的分流完成后，**停止，等待指令**再处理下一个目录。NEVER 跨目录连续执行。

## 注意事项

- **NEVER** 以关键字搜索代替语义判断；关键字最多只能用于辅助定位案例，不能作为分类依据。
- **NEVER** 修改原句的文字内容——只做移动和删除，不改写。
- **NEVER** 将同一句子或句组写入多个目标文件。
- Expert.md、Tools-TodoList.md、Tools.md、Emotional.md 和 Constraints.md **不参与** L1–L9 的语言审计（它们是原材料分拣产物，不是 Skill 文档）。
- 若整个目录无某一类内容，**不创建**对应空文件。
