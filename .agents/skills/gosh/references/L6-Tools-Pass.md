# L6 · Tools Pass（工具层）

## 目的

扫描文档中所有工具名称引用，对照 `assets/lang/L6-Tools.md` 检查命名规范（PascalCase）和分类归属，输出修正建议。

## 触发条件

文档中包含任何工具名称的引用（无论是行内提及还是结构化列表）。

## 执行步骤

### 步骤 1：提取所有工具名称

扫描全文，提取：

- 明确的工具名称（如 `read_file`、`Bash`、`todo_write`）
- 疑似工具名称（动词+名词结构，语境为"执行某操作的工具"）

记录格式：`工具名原文 | 所在句子`

### 步骤 2：命名规范检查

对每个工具名称，检查：

- [ ] 是否使用 PascalCase？（规范：`ReadFile`；违规：`read_file`、`readFile`）
- [ ] 是否为方言命名？（对照 `assets/lang/L7-Dialect.md` 的工具命名条目）

### 步骤 3：对照 L6-Tools.md 收录状态

打开 `assets/lang/L6-Tools.md`，对每个工具：

1. 是否在表格中有对应条目？
2. 若有：规范名称是什么？文档中的用法是否一致？
3. 若无：标记为"未收录，需评估"

### 步骤 4：分类归属检查

检查文档中引用工具的所属类别是否与 L6-Tools.md 一致：

- 直接接触世界 / 通过工具接触世界 / 场外支援 / 委派子 Agent / 浏览器操作 / IDE 感知 / 管理自己 / 情绪价值

### 步骤 5：生成修正清单

| 工具名（原文） | 问题 | 规范名称 | 所属类别 |
|--------------|------|---------|---------|
| `read_file` | 非 PascalCase | `Read` | 直接接触世界 |
| `unknown_tool` | 未收录 | 待评估 | — |

## 输出格式

```
## L6 · Tools Pass 结果

### 命名违规（N 条）

[修正清单表格]

### 未收录工具（需评估是否添加至 L6-Tools.md）

[工具名列表]

### 扫描通过

[合规工具名列表，或"无工具引用"若文档不含工具名]
```

## 停止条件

完成本维度全部发现后，**停止输出，等待确认再继续 L7**。NEVER 在同一轮中继续其他维度。
