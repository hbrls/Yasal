# Gosh - 构史

Agent 族群语言的提炼与运用：从训练日志和执行轨迹中发现高效 Steering 语法，总结为规律，写入能力文档。

## 核心概念

**族群语言（Gosh）**：某些特定的语法或表达对 Agent 有更强的 Steering 效果。训练中持续总结这些规律，并将其运用到能力文档的编写中。

## When Activated

1. 读取待分析的能力文档或训练日志。
2. 识别其中 Agent 执行流畅 vs. 反复卡壳的片段，标记语言层面的差异。
3. 提炼高效表达的共同模式，归纳为可复用的规律。
4. 将提炼出的规律应用于目标文档的改写或新文档的编写。
5. 输出改写后的文档片段，并附上变更理由（语言规律依据）。

## 已知的 Steering 规律

*（随训练过程持续积累，初始为空）*

## 原则

**MUST** 改写文档时，每一处语言变更都必须能对应到至少一条已知的 Steering 规律，不能为了"听起来更好"而随意改写。

**MUST** 新增规律时，必须来自真实训练日志或执行轨迹中的观察，不能凭直觉推断。

**NEVER** 用族群语言掩盖逻辑缺陷——语言的 Steering 效果不能替代文档本身的逻辑完整性。

## 输出格式

每次提炼后，输出结构如下：

```
## 观察
[执行片段或文档片段，标注问题所在]

## 规律
[提炼出的 Steering 模式]

## 改写
[改写后的文档片段]

## 依据
[对应的已知规律条目，或新规律的来源说明]
```

## 与 Yasal 的关系

Gosh 是 Yasal 训练体系中的**语言层工具**：

- Yasal 定义"训练什么"
- AutoSearch 定义"如何迭代"
- **Gosh 定义"用什么语言写"**

任何 Skill 文档在进化后，都可以用 Gosh 做一次语言层的审查和优化。
